# User Security: Password & 2FA (OTP) — UI/UX Design Spec (v1.1)

## 1. Security Settings Page (Dashboard)
**Path**: `/dashboard/security`

### Section B: Two-Factor Authentication (2FA) - Enhanced Flow
**Interaction Pattern**: Multi-step Setup Wizard (or Modal).

#### Step 1: Bind Authenticator
- **Instruction**: "Scan this QR code with your Authenticator app."
- **QR Area**: Use a square container with a subtle `indigo-50` background. Render QR using `canvas` or SVG.
- **Manual Entry**: Display the Secret Key clearly for users who can't scan.

#### Step 2: Verification
- **Instruction**: "Enter the 6-digit code generated by your app."
- **Input**: 6-digit numeric field with immediate validation.
- **Action**: "Verify & Continue" button.

#### Step 3: Secure Recovery Codes (The "Check-out" step)
- **Concept**: Users MUST see their codes before the setup is complete.
- **Display**: A 2x5 grid of 10 codes.
- **Mandatory Action**:
  - "Download codes (.txt)" OR "Copy all codes" button.
  - A checkbox: "I have securely saved these recovery codes."
- **Completion**: "Enable 2FA Now" button becomes active only after the checkbox is ticked.

### Section C: 2FA Status & Management
Once enabled, the card displays:
- **Status**: `Badge(emerald-600)` "2FA Enabled".
- **Info**: "You have 10 unused recovery codes."
- **Actions**:
  - "View / Regenerate Recovery Codes" (Requires current password).
  - "Disable 2FA" (Requires current password + confirmation).
